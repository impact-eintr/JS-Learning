# Git
## 版本控制
- 实现跨区域多人协同开发
- 追踪和记载一个或多个文件的历史记录
- 组织和保护你的源代码和文档
- 统计工作量
- 并行开发,提高开发效率
- 跟踪记录制衡个软件的开发过程
- 减轻开发人员的负担，节约时间，同时降低人为错误

> `git`就是最流行的分布式版本控制

所有的版本信息本仓库全部同步到本地的每一个用户，这样就可以在本地查看所有的版本历史，可以离线在本地提交，只需要在联网时`push`到对应的服务器或者其他用户哪里，由于每个用户哪里保存的都是所有的版本数据，只要有一个用户的设备没有问题，就可以回复所有的数据，但是这也增加了存储空间的占用
